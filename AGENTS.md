## Architecture

```

Browser (speaker PCM 24kHz) â—€â”€â”€WSâ”€â”€  FastAPI relay â—€â”€â”€WSâ”€â”€  Qwen Omni S2S
```

- **One model does everything**: `qwen3.5-omni-plus-realtime` handles STT + LLM + TTS in a
  single WebSocket. There is no separate STT/LLM/TTS â€” that's why latency is low.
- FastAPI is a pure relay: bridges browser binary PCM frames â†” Qwen JSON+base64 protocol.

## Voice cloning (KEY FACTS â€” verified working)

- A cloned `voice_id` works **directly** in the Omni session â€” just pass it as the session
  `voice` field. No separate TTS pipeline. `omni.py` needs zero changes for custom personas.
- Enrollment is ONE synchronous HTTP call, returns a reusable `voice_id`:
  - `POST https://dashscope-intl.aliyuncs.com/api/v1/services/audio/tts/customization`
  - body: `{model:"qwen-voice-enrollment", input:{action:"create", target_model:"qwen3.5-omni-plus-realtime", preferred_name:<slug>, audio:{data:"data:<mime>;base64,<...>"}}}`
  - **Gotchas**: `preferred_name` is REQUIRED (lowercase alphanumeric); `audio` must be a nested
    object `{data: <dataURI>}`, NOT a flat string; response voice_id at `output.voice`.
- No transcript needed for Qwen enrollment (unlike the local mlx-audio path).

## Personas: instructions, clip limits, session scoping

- **Instructions DO work** â€” the Omni `instructions` field steers persona behavior well, but
  needs forceful wording. "Be rude, interrupt, mock me" â†’ genuinely rude; "be a bit tough" â†’ mild.
  Verified via `INSTRUCT="..." uv run python test_omni_direct.py`.
- **Voice clip limits** (Qwen enrollment): 10â€“20s ideal, 60s / 10MB hard max. Frontend trims BOTH
  recordings (auto-stop at 30s) and uploads (`lib/audioTrim.ts` decodes any format â†’ mono WAV,
  first 30s) so a user can't submit an hour-long podcast. Backend also rejects >10MB.
- **Custom personas are per browser session, NOT global.** Frontend generates a UUID in
  localStorage (`lib/session.ts`) and sends it as `X-Session-Id` on every persona request.
  Backend scopes `list`/`delete` by `session_id` (column on the personas table). Presets are
  global. The WS `/converse/{id}` looks up by id alone (you already hold the id).

## Turn-state UX + barge-in (Phase 4)

- The WS now carries TWO frame types: **binary** = audio PCM; **text** = control state
  `{"type":"state","event":"<qwen event>"}`. `omni.py:receive_events()` yields tagged tuples
  `("audio", bytes)` / `("event", type)`; `main.py` forwards them as bytes / text respectively.
- Forwarded events drive the frontend phase machine: `speech_started`â†’listening,
  `speech_stopped`/`response.created`â†’thinking, first audioâ†’speaking, `response.done`/
  `response.audio.done`â†’listening.
- **Barge-in**: client-side VAD (mic RMS > `VAD_LEVEL` in the converse page) while phase==speaking
  â†’ `usePlayback.flush()` (stops/clears scheduled audio, keeps ctx open) + WS text
  `{"type":"barge_in"}` â†’ backend calls `session.cancel_response()` (`{"type":"response.cancel"}`).
  Browserâ†’backend now uses `websocket.receive()` and branches bytes vs text.
- **Echo**: assumes HEADPHONES (no AEC). On speakers the AI interrupts itself. Future: energy gating / AEC.
- **Two modes** (chosen on idle screen, sent as WS `?mode=`):
  - `ptt` (push-to-talk, DEFAULT, reliable): backend uses `turn_detection=None`. Frontend only
    streams mic audio while the talk button / Spacebar is held; on release sends WS `{"type":"commit"}`
    â†’ backend `session.commit_and_respond()` (`input_audio_buffer.commit` + `response.create`).
    No semantic_vad guessing, no false barge-in, works on speakers. Phase `ready` = waiting for press.
  - `hands_free` (experimental): `semantic_vad` + client-VAD barge-in (the Phase-4 behavior). Flaky
    on bare mic+speaker because VAD must GUESS turn boundaries; needs headphones.
- Why hands-free is flaky even with headphones: semantic_vad guesses when you finished (misses pauses/
  soft speech â†’ no reply) and any noise during AI speech can trip client-VAD â†’ false `response.cancel`.
  ptt removes the guessing entirely.
- **Interrupt gotcha (fixed):** flushing playback isn't enough â€” after barge-in, Qwen keeps sending
  buffered `response.audio.delta` chunks for a beat, and re-enqueuing them made the AI "finish its
  sentence." Fix = `acceptAudioRef` gate in the converse page: `interrupt()` sets it false (+ flush +
  barge_in); `handleAudio` drops chunks while false; only `response.created` reopens it. PTT press
  during speaking/thinking calls `interrupt()`.
- `omni.py` error events are now logged (warning) NOT raised â€” a stray `response.cancel` error must
  not kill the session.
- **Orb** (`components/ui/voice-powered-orb.tsx`, ogl/WebGL): prop-driven â€” `state` + `getLevel()`.
  It NEVER opens its own mic (the supplied version did; that was stripped). `getLevel` returns
  mic level when listening, `usePlayback.getLevel()` (AnalyserNode RMS) when speaking. Needs `ogl`
  + `lib/utils.ts` `cn()`. `ConversationControls.tsx` is now unused (controls inlined in the page).

## Key protocol facts

- Qwen `response.audio.delta` event: audio base64 is at `event["delta"]` (a STRING, not `delta.audio`).
- Omni needs trailing silence to trigger end-of-turn VAD when feeding a fixed clip (real mic streams silence naturally).
- Audio: browserâ†’Qwen 16kHz PCM, Qwenâ†’browser 24kHz PCM, both 16-bit mono.

## Per-user model + hardening (Phase 6)

- **Privacy**: custom personas are scoped by `session_id` everywhere â€” `list`/`delete` (header
  `X-Session-Id`) AND the WS `/converse` (query `?sid=`). A custom persona can ONLY be used by its
  creating session (WS closes 4003 otherwise). Preset personas are open. `session_id` = a browser
  (localStorage UUID), NOT an account â€” clearing storage loses personas. Real accounts = future work.
- **Abuse/cost guards** (`backend/limits.py`, in-memory; single-instance only): concurrent-WS cap
  per session, max conversation duration (auto-close), daily enrollment cap. Move to Redis if you
  run multiple instances.
- **DB**: SQLite is fine on ONE box. Multiple backend instances behind a load balancer â†’ switch to
  managed Postgres (each instance otherwise has its own file).
